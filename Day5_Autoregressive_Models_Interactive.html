🎯Motivation — Why Do We Need This?
The curse of dimensionality and the goal of generative modeling
💡

Core Problem: We want to estimate distributions of complex, high-dimensional data. A single 128×128 RGB image has 128 × 128 × 3 = 49,152 dimensions. Directly modeling such a space is practically impossible!

⚡

Computational Efficiency

Fast training convergence, reasonable resource usage, compact model size.

📊

Statistical Efficiency

Expressiveness to capture complex relationships + generalization to unseen data.

🎯

Performance

High quality sampling & speed. Good compression rate and speed.

📐Fundamentals — The Core Idea
How autoregressive models work: chain rule factorization
🔑

Core Idea: Factorize the joint distribution using the chain rule of probability, expressing it as a product of conditional probabilities.

The Autoregressive Factorization (Chain Rule)
\[ p(x_1, x_2, \ldots, x_n) = \prod_{i=1}^{n} p(x_i \mid x_1, x_2, \ldots, x_{i-1}) = \prod_{i=1}^{n} p(x_i \mid x_{

This allows us to model complex data one element (or "token") at a time, conditioned on previously generated elements.

🎮 Interactive: See the Chain Rule in Action

Hover over each conditional — they multiply together to give you the full joint probability!

p(x) = p(x₁) × p(x₂|x₁) × p(x₃|x₁,x₂) × p(x₄|x₁,x₂,x₃) × … × p(xₙ|x₁,…,xₙ₋₁)
🔎

Notice how each term conditions on everything that came before it. This is why it's called autoregressive — the model regresses on its own previous outputs.

Why does this solve the curse of dimensionality?

Instead of estimating a single massive joint distribution over all 49,152 pixel values simultaneously, we break it down into many smaller, manageable conditional distributions. Each conditional just needs to model one variable given the previous ones.

Design Considerations

🔤 Tokenization ▼

Discretizing continuous data to form sequences. Examples:

  • Images: split into patches, or model pixel-by-pixel (0–255 as 256 categories)
  • Audio: discretize waveform samples (e.g., 256 levels with μ-law encoding)
  • Text: naturally discrete — words, subwords, or bytes
💾 Caching (KV Cache) ▼

Essential for efficient inference. During generation, previously computed hidden states/keys/values are cached so we don't re-compute them for every new token. Without caching, each new generated element would require re-processing all previous elements — O(n²) cost instead of O(n).

🏛️ Decoder-only vs Encoder-Decoder ▼

Decoder-only (e.g., GPT): Great for pure generation — each token attends only to past tokens via causal masking.

Encoder-Decoder (e.g., T5): Better for conditioning on an input (translation, summarization). Encoder processes full input bidirectionally; decoder generates autoregressively.

📉Learning Autoregressive Models
KL Divergence, Expected Log-Likelihood, and Maximum Likelihood Estimation

The goal of learning: return a model M̂ that precisely captures the data distribution Pdata.

⚠️

This is not exactly achievable because: (1) we only have limited data — a rough approximation, (2) computational constraints. For MNIST, 2784 ≈ 10236 possible images, but we have ~107 examples — extremely sparse coverage!

KL Divergence — Measuring Closeness

Kullback-Leibler Divergence
\[ D(P_{data} \| P_\theta) = \mathbb{E}_{x \sim P_{data}} \left[ \log \frac{P_{data}(x)}{P_\theta(x)} \right] = \sum_x P_{data}(x) \log \frac{P_{data}(x)}{P_\theta(x)} \]
  • D = 0 iff the two distributions are identical
  • Measures the "compression loss" (in bits) of using Pθ instead of Pdata
  • Note: KL divergence is not symmetric — D(P‖Q) ≠ D(Q‖P)

From KL to Log-Likelihood

KL Divergence Decomposition
\[ D(P_{data} \| P_\theta) = \underbrace{\mathbb{E}_{x \sim P_{data}}[\log P_{data}(x)]}_{\text{constant w.r.t. } \theta} - \mathbb{E}_{x \sim P_{data}}[\log P_\theta(x)] \]
💡

The first term doesn't depend on θ! So minimizing KL divergence ≡ maximizing the expected log-likelihood:

Equivalence
\[ \arg\min_{P_\theta} D(P_{data} \| P_\theta) = \arg\max_{P_\theta} \mathbb{E}_{x \sim P_{data}}[\log P_\theta(x)] \]

Maximum Likelihood Estimation (MLE)

Since we don't know Pdata, we approximate the expected log-likelihood with the empirical log-likelihood over our dataset D:

MLE Objective
\[ \arg\max_{P_\theta} \frac{1}{|D|} \sum_{x \in D} \log P_\theta(x) \]
⚠️ Caveat: Because we ignore H(Pdata), we don't know how close we are to the true optimum. Also, Pdata is unknown in general — we only have samples from it. In practice, because of the log, samples where Pθ(x) ≈ 0 contribute heavily to the loss, pushing the model to assign non-zero probability everywhere.
🔵FVSBN — Fully Visible Sigmoid Belief Network
Bengio & Bengio (2000) — The pioneering neural autoregressive model
Year: 2000 Binary Data Sigmoid Units No Latent Variables

Key Idea

A Bayesian network defined over observed variables only (no hidden/latent variables). Each conditional distribution is modeled using a sigmoid (logistic) unit. Each variable is conditionally independent of all others given its predecessors.

Formulation

Joint Probability
\[ p(x) = \prod_{i=1}^{n} p(x_i \mid x_{
Each Conditional (Sigmoid Parameterization)
\[ p(x_i = 1 \mid x_{
Full Parameterization (for MNIST example)
\[ \hat{x}_i = P(X_i=1 \mid x_1,\ldots,x_{i-1};\, \alpha_i) = \sigma\!\left(\alpha_0^i + \sum_{j=1}^{i-1} \alpha_j^i x_j\right) \]

How to Sample New Images

  1. Sample x₁ ~ P(x₁) using np.random.choice([1,0], p=[x̂₁, 1−x̂₁])
  2. Sample x₂ ~ P(x₂ | X₁ = x₁)
  3. Sample x₃ ~ P(x₃ | X₁ = x₁, X₂ = x₂)
  4. … continue sequentially until all n variables are sampled
🔢

Parameter Count: Total = 1 + 2 + 3 + … + n = n(n+1)/2. For 784 MNIST pixels: ~307,000 parameters. Each variable xi gets its own weight vector.

✅ Pros
  • Likelihood is tractable (can be computed exactly)
  • Simple formulation — easy to understand
  • Foundational — established neural AR density estimation
❌ Cons
  • Poor scalability: quadratic parameter growth O(n²)
  • Each variable has its own independent weight vector
  • No weight sharing between variables
  • Shallow model — limited expressiveness

🔢Non-Binary Outputs — Going Beyond 0/1
Categorical distributions for discrete pixel values (0–255)
❓

What if we want to predict a pixel value between 0 and 255? Binary outputs can't do this. We need to extend to categorical distributions.

Categorical Approach

Hidden Representation (same as NADE)
\[ h_i = \sigma(W_{:,
Categorical Distribution over K classes
\[ P(x_i \mid x_1, \ldots, x_{i-1}) = \text{Cat}(p_1^i, p_2^i, \ldots, p_K^i) \] \[ \hat{x}_i = (p_1^i, \ldots, p_K^i) = \text{softmax}(A_i h_i + b_i) \]

🔄Recurrent Neural Networks (RNNs)
Handling growing histories with a recursive hidden state
🤔

Challenge: To model P(xt | x1:t-1), the "history" x1:t-1 keeps getting longer and longer. You can't have a fixed-size input!

The RNN Solution

Keep a summary (hidden state ht) and recursively update it:

RNN Recurrence
\[ h_t = f(h_{t-1}, x_t) \qquad \text{(update hidden state)} \] \[ P(x_{t+1} \mid x_{1:t}) = g(h_t) \qquad \text{(predict next element)} \]

The hidden state ht acts as a compressed representation of all previous inputs. The same functions f and g are applied at every step (weight sharing over time).

RNN Use Cases

📸

Image Captioning

Many-to-one: CNN encodes image → RNN generates caption word by word.

✍️

Text Generation

One-to-many or many-to-many: Generate a sequence of text tokens autoregressively.

🤖

Chatbots

Seq-to-seq with encoder-decoder RNN architecture for question-answer pairs.

✅ Pros
  • Works on sequences of arbitrary length
  • Very general: for every computable function, there exists a finite RNN that computes it (Turing complete)
❌ Cons
  • Requires an ordering of elements
  • Sequential likelihood evaluation — very slow training
  • Sequential generation (unavoidable in AR models)
  • Vanishing/exploding gradients (mitigated by LSTM/GRU)

Architecture Details

1️⃣ Row LSTM ▼

Applies a 1D LSTM across each row. Information flows left-to-right within each row. Maintains the autoregressive structure by only conditioning on pixels above and to the left.

The hidden state captures the "row context" and is passed to the next pixel in the row.

2️⃣ Diagonal BiLSTM ▼

Applies a 2D LSTM diagonally across the image. Allows pixels to be conditioned on a broader context from multiple directions. Captures longer-range dependencies than Row LSTM but is more computationally expensive.

3️⃣ Masked Convolutions ▼

Prevent the model from "seeing future" pixels. Each convolution kernel is masked so that only pixels above and to the left contribute — preserving the autoregressive property.

For RGB images: In the first layer, R channel can attend to context only; G channel can attend to context + R; B channel can attend to context + R + G. In later layers, channels attend to themselves too.

Training and Inference

  • Training: Maximum Likelihood Estimation (MLE) — maximize joint probability of pixels in training images. Training can be parallelized (teacher forcing).
  • Inference/Sampling: Sequential — start from blank canvas, predict pixel by pixel. Cannot be parallelized.
✅ Pros
  • High-quality images with complex structures
  • Captures long-range dependencies
  • Tractable likelihood computation
❌ Cons
  • Very slow inference (sequential pixel-by-pixel)
  • Computationally intensive
  • Slower than PixelCNN for generation

How the mask works:

Multiplying the kernel by this binary mask before applying convolution enforces the autoregressive property.

⚠️ The Blind Spot Problem

🚨 Known Issue: PixelCNN's masked convolutions create a blind spot in the receptive field. The model cannot see pixels to the right and below the current pixel — which means after several layers, some context is always missing! This can lead to artifacts in generated images.

Fix: PixelSNAIL (2017) addresses this by incorporating self-attention mechanisms, giving the model access to the full causal context without the blind spot issue.

⚡Transformer-based Autoregressive Models
Vaswani et al. (2017) — Self-attention for flexible global context
🚀

Transformers replaced recurrence with self-attention, enabling fully parallel processing during training while still generating autoregressively at inference time via causal masking.

How Causal Self-Attention Works

In a standard Transformer, each position can attend to all positions. In an autoregressive (causal) Transformer, a lower-triangular mask is applied so position i can only attend to positions ≤ i:

Causal Self-Attention
\[ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}} + M\right) V \] where \(M_{ij} = -\infty\) if \(j > i\) (mask out future positions), \(0\) otherwise.

📝Summary — The Big Picture
Everything you need to remember from Day 5

What Autoregressive Models Do

1️⃣

Define an Ordering

Choose an ordering for variables: x₁, x₂, …, xₙ (e.g., raster scan for images, left-to-right for text).

2️⃣

Factorize the Joint

Express joint as a product of conditionals via the chain rule: P(X) = ∏ P(xᵢ | x<ᵢ).

3️⃣

Train with MLE

Maximize log-likelihood of training data — equivalent to minimizing KL divergence from the true data distribution.

4️⃣

Sample Recursively

Generate x₁ first, then x₂|x₁, then x₃|x₁,x₂, and so on. Always sequential at generation time.

Key Properties

✅

Tractable Likelihoods: Unlike GANs or VAEs, autoregressive models give exact log-likelihood — great for comparing models and data compression.

✅

Flexible: Extends naturally to multi-class, multi-dimensional, and structured data. Works for text, images, audio, video.

✅

Efficient Training: Modern architectures (PixelCNN, Transformers) allow parallelized training with teacher forcing.

Key Limitations

  • ⚠️ No global latent features: No natural way to extract a compact semantic representation of the data.
  • ⚠️ Not suited for clustering: Not designed for unsupervised feature learning or clustering tasks.
  • ⚠️ Ordering sensitivity: The choice of variable ordering can significantly affect model performance.
  • ⚠️ Sequential generation: Always sequential at inference time — cannot be fully parallelized.

The Appendix Extras: Learning from Histograms

📊 Learning by Counting (Histogram Approach) ▼

The simplest generative model: estimate Pdata by counting frequencies in training data. For a k-valued variable: pi = (# times value i appears) / (total data points).

Problem: learned histogram = training data distribution → poor generalization. The model just memorizes the training set.

Solution: Parameterized distributions (e.g., Gaussian, mixture models, neural networks) that can generalize beyond training examples — the key motivation for all the models in this lecture.
