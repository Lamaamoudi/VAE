🎯Motivation — Why Do We Need This?
The curse of dimensionality and the goal of generative modeling
💡

Core Problem: We want to estimate distributions of complex, high-dimensional data. A single 128×128 RGB image has 128 × 128 × 3 = 49,152 dimensions. Directly modeling such a space is practically impossible!

⚡

Computational Efficiency

Fast training convergence, reasonable resource usage, compact model size.

📊

Statistical Efficiency

Expressiveness to capture complex relationships + generalization to unseen data.

🎯

Performance

High quality sampling & speed. Good compression rate and speed.

✍️

Text Generation

Large Language Models (GPT), RNNs — generate realistic text word by word.

🖼️

Image Generation

PixelCNN, PixelRNN — synthesize realistic images pixel by pixel.

🎵

Audio Synthesis

WaveNet — generate natural-sounding speech and music sample by sample.

🔍

Anomaly Detection

Identify data that deviates from the learned distribution.

💡 MNIST Example: Each digit is a 784-dimensional binary vector. Total possible states = 2784 ≈ 10236. Even with 107 training examples, coverage is extremely sparse! We need smart models, not brute-force tables.
📐Fundamentals — The Core Idea
How autoregressive models work: chain rule factorization
🔑

Core Idea: Factorize the joint distribution using the chain rule of probability, expressing it as a product of conditional probabilities.

The Autoregressive Factorization (Chain Rule)
\[ p(x_1, x_2, \ldots, x_n) = \prod_{i=1}^{n} p(x_i \mid x_1, x_2, \ldots, x_{i-1}) = \prod_{i=1}^{n} p(x_i \mid x_{<i}) \]

This allows us to model complex data one element (or "token") at a time, conditioned on previously generated elements.

🎮 Interactive: See the Chain Rule in Action

Hover over each conditional — they multiply together to give you the full joint probability!

p(x) = p(x₁) × p(x₂|x₁) × p(x₃|x₁,x₂) × p(x₄|x₁,x₂,x₃) × … × p(xₙ|x₁,…,xₙ₋₁)
🔎

Notice how each term conditions on everything that came before it. This is why it's called autoregressive — the model regresses on its own previous outputs.

Why does this solve the curse of dimensionality?

Instead of estimating a single massive joint distribution over all 49,152 pixel values simultaneously, we break it down into many smaller, manageable conditional distributions. Each conditional just needs to model one variable given the previous ones.

Graphical models encoding conditional dependencies. Classic statistical approach.

🔄

Naturally suited for sequential data. Keep a hidden state summary updated recursively.

🎭

Feedforward network with masks to enforce the autoregressive property efficiently.

🏗️

PixelCNN, WaveNet — use masked convolutions so future elements can't influence current ones.

🔭

Attention-based

Transformers with causal masks — enable flexible context modeling with global dependencies.

Design Considerations

🔤 Tokenization ▼

Discretizing continuous data to form sequences. Examples:

  • Images: split into patches, or model pixel-by-pixel (0–255 as 256 categories)
  • Audio: discretize waveform samples (e.g., 256 levels with μ-law encoding)
  • Text: naturally discrete — words, subwords, or bytes
💾 Caching (KV Cache) ▼

Essential for efficient inference. During generation, previously computed hidden states/keys/values are cached so we don't re-compute them for every new token. Without caching, each new generated element would require re-processing all previous elements — O(n²) cost instead of O(n).

🏛️ Decoder-only vs Encoder-Decoder ▼

Decoder-only (e.g., GPT): Great for pure generation — each token attends only to past tokens via causal masking.

Encoder-Decoder (e.g., T5): Better for conditioning on an input (translation, summarization). Encoder processes full input bidirectionally; decoder generates autoregressively.

📉Learning Autoregressive Models
KL Divergence, Expected Log-Likelihood, and Maximum Likelihood Estimation

The goal of learning: return a model M̂ that precisely captures the data distribution Pdata.

⚠️

This is not exactly achievable because: (1) we only have limited data — a rough approximation, (2) computational constraints. For MNIST, 2784 ≈ 10236 possible images, but we have ~107 examples — extremely sparse coverage!

KL Divergence — Measuring Closeness

Kullback-Leibler Divergence
\[ D(P_{data} \| P_\theta) = \mathbb{E}_{x \sim P_{data}} \left[ \log \frac{P_{data}(x)}{P_\theta(x)} \right] = \sum_x P_{data}(x) \log \frac{P_{data}(x)}{P_\theta(x)} \]
  • D = 0 iff the two distributions are identical
  • Measures the "compression loss" (in bits) of using Pθ instead of Pdata
  • Note: KL divergence is not symmetric — D(P‖Q) ≠ D(Q‖P)

From KL to Log-Likelihood

KL Divergence Decomposition
\[ D(P_{data} \| P_\theta) = \underbrace{\mathbb{E}_{x \sim P_{data}}[\log P_{data}(x)]}_{\text{constant w.r.t. } \theta} - \mathbb{E}_{x \sim P_{data}}[\log P_\theta(x)] \]
💡

The first term doesn't depend on θ! So minimizing KL divergence ≡ maximizing the expected log-likelihood:

Equivalence
\[ \arg\min_{P_\theta} D(P_{data} \| P_\theta) = \arg\max_{P_\theta} \mathbb{E}_{x \sim P_{data}}[\log P_\theta(x)] \]

Maximum Likelihood Estimation (MLE)

Since we don't know Pdata, we approximate the expected log-likelihood with the empirical log-likelihood over our dataset D:

MLE Objective
\[ \arg\max_{P_\theta} \frac{1}{|D|} \sum_{x \in D} \log P_\theta(x) \]
⚠️ Caveat: Because we ignore H(Pdata), we don't know how close we are to the true optimum. Also, Pdata is unknown in general — we only have samples from it. In practice, because of the log, samples where Pθ(x) ≈ 0 contribute heavily to the loss, pushing the model to assign non-zero probability everywhere.
🔵FVSBN — Fully Visible Sigmoid Belief Network
Bengio & Bengio (2000) — The pioneering neural autoregressive model
Year: 2000 Binary Data Sigmoid Units No Latent Variables

What Does the Name Mean?

"Fully Visible" = every variable in the model is directly observed — no hidden or latent variables. "Sigmoid Belief Network" = each conditional probability is computed by a sigmoid (logistic) function applied to a linear combination of the previous variables. Think of it as running a tiny logistic regression at every single pixel, using all prior pixels as features.

📊 Network Structure — 4 Variables

For n=4 variables, here is what FVSBN looks like. Notice how each node has its own unique set of weights, and the number of weights grows with each variable:

💡 Key insight: Variable xᵢ has exactly i parameters (α₀ⁱ, α₁ⁱ, …, αᵢ₋₁ⁱ). Total = 1+2+3+…+n = n(n+1)/2. For MNIST (n=784): that's 307,580 parameters — and growing quadratically with image size. This is FVSBN's fatal weakness.

The Formula

Joint Probability — Product of Conditionals
\[ p(x_1, x_2, \ldots, x_n) = \prod_{i=1}^{n} p(x_i \mid x_1, \ldots, x_{i-1}) \]
Each Conditional — Sigmoid Parameterization
\[ p(x_i = 1 \mid x_1, \ldots, x_{i-1}) = \sigma\!\left(\alpha_0^i + \sum_{j=1}^{i-1} \alpha_j^i \, x_j \right) \]

Where: σ(z) = 1/(1+e−z) is the sigmoid function (squashes any number to [0,1]). α₀ⁱ is the bias for variable xᵢ. αⱼⁱ is the weight measuring how much xⱼ influences xᵢ. Since data is binary, P(xᵢ = 0) = 1 − P(xᵢ = 1).

🔢 Worked Numerical Example (3 Pixels)

Let's trace through a tiny MNIST example: 3 binary pixels with true values (x₁=1, x₂=1, x₃=0). We want to evaluate how likely this image is under the model.

📌 Learned weights for this example:

α₀¹ = 0.5   |   α₀² = 0.3, α₁² = 0.8   |   α₀³ = −0.2, α₁³ = 0.6, α₂³ = 0.4

Step 1 P(x₁=1) = σ(0.5) = 1/(1+e⁻⁰·⁵) = 0.622
Step 2 P(x₂=1 | x₁=1) = σ(0.3 + 0.8×1) = σ(1.1) = 0.750
Step 3 P(x₃=0 | x₁=1, x₂=1) = 1 − σ(−0.2 + 0.6×1 + 0.4×1) = 1 − σ(0.8) = 1 − 0.690 = 0.310
Joint P P(x₁=1, x₂=1, x₃=0) = 0.622 × 0.750 × 0.310 = 0.145

📈 Training maximizes log(0.622) + log(0.750) + log(0.310) = −0.476 − 0.288 − 1.171 = −1.935. The optimizer pushes this value toward 0 by adjusting all α weights.

How to Sample New Images

  1. Compute x̂₁ = σ(α₀¹). Sample x₁ ∈ {0,1} with P(x₁=1) = x̂₁. In code: x1 = np.random.choice([1,0], p=[x_hat1, 1-x_hat1])
  2. Plug x₁ into x̂₂ = σ(α₀² + α₁²·x₁). Sample x₂.
  3. Plug x₁, x₂ into x̂₃ = σ(α₀³ + α₁³·x₁ + α₂³·x₂). Sample x₃.
  4. Continue until all n pixels are sampled — each depends on all previous ones.
✅ Pros
  • Exact tractable likelihood — no approximations
  • Conceptually simple — logistic regression per variable
  • Historic — first neural AR density estimation model
❌ Cons
  • O(n²) parameters — scales quadratically, unusable for large images
  • No weight sharing — xᵢ's weights tell us nothing about xⱼ's weights
  • Binary only — can't predict pixel values 0–255
  • Only logistic regression — not expressive enough for complex patterns
🟢NADE — Neural Autoregressive Distribution Estimator
Larochelle & Murray (2011) — Scalable version via weight sharing
Year: 2011 Weight Sharing Masked Neural Net Binary Data
🚀

Key Innovation: Replace FVSBN's independent weight vectors with a single shared weight matrix W reused across all conditionals — cuts parameters from O(n²) down to O(d·H) while adding a hidden layer for more expressive power.

The Problem With FVSBN

Recall that FVSBN needs a different set of weights for every variable. To predict x3, it uses weights w3. To predict x100, it uses completely different weights w100. For d=784 pixels each with their own weight vector, that's 307,000+ parameters — and none of the network's "knowledge" transfers from predicting x3 to predicting x100.

🔵 FVSBN — Unique Weights

Every conditional has its own weight vector:

w₁ (for x₁)
w₂,₁ (for x₂) w₂,₂
w₃,₁ w₃,₂ w₃,₃
Each row = new weights. Zero sharing. O(n²) params.
🟢 NADE — Shared Matrix W

One matrix W, use more columns each time:

W[:,0] → h₁
W[:,0] W[:,1] → h₂
W[:,0] W[:,1] W[:,2] → h₃
Same columns reused. O(d·H) params. Generalizes better.

Architecture & Formulas

NADE introduces a hidden layer h between input and output. Two steps per variable:

Step 1 — Hidden Representation
\[ \mathbf{h}_i = \sigma\!\bigl(\mathbf{W}_{:,\,1:i-1}\; \mathbf{x}_{1:i-1} + \mathbf{c}\bigr) \]
W is a shared (H × d) matrix — same matrix for all i, just use the first i−1 columns.
c is a shared bias vector (H-dimensional).
hi is an H-dimensional vector — the hidden state for variable i.
Step 2 — Conditional Probability
\[ p(x_i \mid x_{1:i-1}) = \sigma\!\bigl(\mathbf{\alpha}^{(i)} \cdot \mathbf{h}_i + b_i\bigr) \]
α(i) is an output weight vector specific to variable i (H-dimensional).
bi is an output bias for variable i.
These are the only per-variable parameters — much fewer than FVSBN.
🔢 Worked Example — 3 pixels, H=2 hidden units

Suppose d=3 pixels, H=2 hidden units. The shared weight matrix W is 2×3:

\[ W = \begin{bmatrix} 0.5 & -0.3 & 0.8 \\ 1.0 & 0.6 & -0.4 \end{bmatrix}, \quad c = \begin{bmatrix} 0.1 \\ -0.2 \end{bmatrix} \]

Observed pixels: x₁ = 1, x₂ = 0. Want to compute h₃ (for predicting x₃):

Use columns 1 & 2 of W: \[ W_{:,1:2} = \begin{bmatrix}0.5 & -0.3 \\ 1.0 & 0.6\end{bmatrix} \]
Linear combination: \[ W_{:,1:2}\,\mathbf{x}_{1:2} + c = \begin{bmatrix}0.5(1)+(-0.3)(0)+0.1 \\ 1.0(1)+0.6(0)+(-0.2)\end{bmatrix} = \begin{bmatrix}0.6 \\ 0.8\end{bmatrix} \]
Apply σ: \[ \mathbf{h}_3 = \sigma\!\left(\begin{bmatrix}0.6 \\ 0.8\end{bmatrix}\right) = \begin{bmatrix}0.646 \\ 0.690\end{bmatrix} \]
With α⁽³⁾=[1.2, −0.5], b₃=0.3: \( p(x_3{=}1 \mid x_1{=}1, x_2{=}0) = \sigma(1.2(0.646) + (-0.5)(0.690) + 0.3) \)
\( = \sigma(0.775 - 0.345 + 0.300) = \sigma(0.730) \approx \mathbf{0.675} \)

Notice: if we also needed h₂, we'd reuse the same W — just the first column W[:,1]. That's weight sharing in action.

🔢Non-Binary Outputs — Going Beyond 0/1
Categorical distributions for discrete pixel values (0–255)
❓

Problem: FVSBN and NADE both produce a single probability: P(xᵢ = 1). That's fine for black-or-white pixels. But real images have pixel values from 0 to 255 — 256 possible levels. Sigmoid can't do that. We need a new output head.

Three Output Regimes

⚫⚪

Binary (K = 2)

Pixel is black or white.
Output: single probability p ∈ [0,1].
Use: Sigmoid

P(x=1) = σ(z)
🎨

Discrete (K = 256)

Pixel is one of 256 intensities.
Output: 256 probabilities summing to 1.
Use: Softmax

(p₀, p₁, …, p₂₅₅) = softmax(Az + b)
〰️

Continuous (K = ∞)

Pixel is a real number.
Output: parameters of a distribution.
Use: Gaussian / GMM

p(x) = N(μ, σ²) → see RNADE

How Softmax Works

Softmax takes a vector of raw scores (logits) and converts them into probabilities. The formula:

Softmax Formula
\[ \text{softmax}(z_k) = \frac{e^{z_k}}{\displaystyle\sum_{j=1}^{K} e^{z_j}} \]
Each raw score zk gets exponentiated (always positive), then normalized so all probabilities sum to 1.
Key property: higher zk → higher P(x=k). It's a differentiable "argmax".

Full NADE for K Classes

Hidden Layer (unchanged from binary NADE)
\[ \mathbf{h}_i = \sigma\!\bigl(\mathbf{W}_{:,\,1:i-1}\;\mathbf{x}_{1:i-1} + \mathbf{c}\bigr) \]
Categorical Output — K classes (K=256 for pixels)
\[ P(x_i = k \mid x_{1:i-1}) = \frac{\exp\!\bigl((\mathbf{A}_i\,\mathbf{h}_i + \mathbf{b}_i)_k\bigr)}{\displaystyle\sum_{j=1}^{K}\exp\!\bigl((\mathbf{A}_i\,\mathbf{h}_i + \mathbf{b}_i)_j\bigr)} \]
Ai is a K×H output matrix (projects hi to K class scores).
bi is a K-dimensional bias vector.
For binary (K=2), this simplifies back to a single sigmoid.

🌊RNADE — Real-valued NADE
Uria et al. (2013) — Continuous data via Mixture of Gaussians
Year: 2013 Continuous Data Mixture of Gaussians
🔬

Problem: Audio samples, pixel intensities, and sensor readings are continuous real numbers — not binary 0/1 or discrete 0–255. Even for pixels: if you just round to 256 bins, you lose quality in gradients and smooth color transitions. RNADE extends NADE to handle xi ∈ ℝ directly.

Why Not Just Use a Single Gaussian?

A single Gaussian can only model a bell-shaped, unimodal distribution. Real-world data is often multimodal — for example, the brightness of a sky pixel depends on whether it's day (bright cluster) or night (dark cluster). A single Gaussian would average them badly:

Green = daytime Gaussian · Blue = nighttime Gaussian · Teal = mixture · Red dashed = single Gaussian attempt

The Fix: Mixture of Gaussians (GMM)

Instead of one Gaussian, use K Gaussians and let the network predict which one to use (mixture weights πk) plus the parameters of each:

RNADE Conditional Density — K Gaussians
\[ p(x_i \mid x_{1:i-1}) = \sum_{k=1}^{K} \pi_k \cdot \mathcal{N}\!\bigl(x_i;\;\mu_k,\;\sigma_k^2\bigr) \]
All three sets of parameters are functions of the context x1:i−1 via the shared NADE hidden layer:

πk — mixture weights: how much Gaussian k contributes. Constrained: πk ≥ 0, Σπk=1 (use softmax).
μk — mean of Gaussian k (center of the bell curve).
σk² — variance of Gaussian k (how wide the bell is). Must be positive (use exp or softplus).

How the Architecture Works

Same NADE hidden layer, but now the output head produces 3K numbers instead of 1:

Step 1 — Hidden Layer (shared, same as NADE)
\[ \mathbf{h}_i = \sigma\!\bigl(\mathbf{W}_{:,1:i-1}\;\mathbf{x}_{1:i-1} + \mathbf{c}\bigr) \]
Step 2 — Output Head produces GMM parameters
\[ (\boldsymbol{\pi}_i,\;\boldsymbol{\mu}_i,\;\boldsymbol{\sigma}_i) = \text{OutputNet}(\mathbf{h}_i) \]
OutputNet is a linear layer followed by: softmax for π, identity for μ, softplus/exp for σ.
Full Joint Distribution
\[ p(\mathbf{x}) = \prod_{i=1}^{d} \sum_{k=1}^{K} \pi_k(\mathbf{x}_{1:i-1})\;\mathcal{N}\!\bigl(x_i;\;\mu_k(\mathbf{x}_{1:i-1}),\;\sigma_k^2(\mathbf{x}_{1:i-1})\bigr) \]
🔢 Worked Example — Audio Sample (K=2 Gaussians)

Suppose the NADE hidden layer outputs parameters for K=2 mixture components for the next audio amplitude xi:

Component 1 (soft sound): \( \pi_1 = 0.7,\quad \mu_1 = 0.1,\quad \sigma_1 = 0.05 \)
Component 2 (loud burst): \( \pi_2 = 0.3,\quad \mu_2 = 0.8,\quad \sigma_2 = 0.15 \)
Query: p(xi = 0.1)? \( p = 0.7 \cdot \mathcal{N}(0.1;\,0.1,\,0.05^2) + 0.3 \cdot \mathcal{N}(0.1;\,0.8,\,0.15^2) \)
= 0.7 × (high peak near 0.1) + 0.3 × (tiny tail far from 0.8) \( \approx 0.7 \times 7.98 + 0.3 \times 0.0003 \approx 5.59 \)
Most of the density is from component 1 (soft sound) — model correctly says "probably a quiet sample".

At training time, the model is trained by maximizing log p(xi) summed over all training samples using gradient descent — exactly like NADE, just with a richer output distribution.

Where RNADE Is Used

🔊

Audio / Speech

Audio waveform samples are real-valued. GMM captures multimodal amplitude distributions (silence vs. voiced sounds).

🌡️

Scientific Data

Sensor readings, temperatures, financial returns — all continuous data where discretization wastes information.

🔗

Precursor to Flows

RNADE directly inspired Masked Autoregressive Flows (MAF) and Inverse Autoregressive Flows (IAF) — key modern generative models.

✅ Pros
  • Handles complex, multimodal continuous distributions
  • Directly models density (unlike GANs)
  • Influential precursor to normalizing flows
❌ Cons
  • Need to pick K (number of Gaussians) as hyperparameter
  • Computationally heavier than discrete NADE
  • Sequential evaluation — slow for high-dimensional data
🔄Recurrent Neural Networks (RNNs)
Handling growing histories with a recursive hidden state
🤔

Challenge: To model P(xt | x1:t-1), the "history" x1:t-1 keeps getting longer and longer. You can't have a fixed-size input!

The RNN Solution

Keep a summary (hidden state ht) and recursively update it:

RNN Recurrence
\[ h_t = f(h_{t-1}, x_t) \qquad \text{(update hidden state)} \] \[ P(x_{t+1} \mid x_{1:t}) = g(h_t) \qquad \text{(predict next element)} \]

The hidden state ht acts as a compressed representation of all previous inputs. The same functions f and g are applied at every step (weight sharing over time).

RNN Use Cases

📸

Image Captioning

Many-to-one: CNN encodes image → RNN generates caption word by word.

✍️

Text Generation

One-to-many or many-to-many: Generate a sequence of text tokens autoregressively.

🤖

Chatbots

Seq-to-seq with encoder-decoder RNN architecture for question-answer pairs.

✅ Pros
  • Works on sequences of arbitrary length
  • Very general: for every computable function, there exists a finite RNN that computes it (Turing complete)
❌ Cons
  • Requires an ordering of elements
  • Sequential likelihood evaluation — very slow training
  • Sequential generation (unavoidable in AR models)
  • Vanishing/exploding gradients (mitigated by LSTM/GRU)
🖼️PixelRNN — Pixel Recurrent Neural Network
van den Oord et al. (2016) — Generating images pixel by pixel
DeepMind 2016 Raster Scan Order LSTM Image Generation
🎯

Core Idea: Model the joint distribution of all pixels by predicting one pixel at a time in raster-scan order (left-to-right, top-to-bottom — like reading text).

PixelRNN Factorization
\[ p(x) = \prod_{i=1}^{n} p(x_i \mid x_1, x_2, \ldots, x_{i-1}) \] For pixel at position (i,j): \[ p(x_{i,j} \mid x_{\text{prev}}) \] where \(x_{\text{prev}}\) = all pixels above row i, plus pixels in row i to the left of column j (raster-scan order).

Why BiLSTM? Two LSTMs scan simultaneously — one from top-left, one from top-right — giving each pixel a richer 2D context. The two hidden states are combined before predicting the pixel value.

Advantage over Row LSTM: Larger effective receptive field (true 2D dependency) at the cost of higher computation. Produces better image quality.

3️⃣ Masked Convolutions ▼

Prevent the model from "seeing future" pixels. Each convolution kernel is masked so that only pixels above and to the left contribute — preserving the autoregressive property.

For RGB images: In the first layer, R channel can attend to context only; G channel can attend to context + R; B channel can attend to context + R + G. In later layers, channels attend to themselves too.

Training and Inference

  • Training: Maximum Likelihood Estimation (MLE) — maximize joint probability of pixels in training images. Training can be parallelized (teacher forcing).
  • Inference/Sampling: Sequential — start from blank canvas, predict pixel by pixel. Cannot be parallelized.
✅ Pros
  • High-quality images with complex structures
  • Captures long-range dependencies
  • Tractable likelihood computation
❌ Cons
  • Very slow inference (sequential pixel-by-pixel)
  • Computationally intensive
  • Slower than PixelCNN for generation

⚠️ The Blind Spot Problem

🚨 Known Issue: PixelCNN's masked convolutions create a blind spot in the receptive field. The model cannot see pixels to the right and below the current pixel — which means after several layers, some context is always missing! This can lead to artifacts in generated images.

Fix: PixelSNAIL (2017) addresses this by incorporating self-attention mechanisms, giving the model access to the full causal context without the blind spot issue.

Transformers replaced recurrence with self-attention, enabling fully parallel processing during training while still generating autoregressively at inference time via causal masking.

How Causal Self-Attention Works

In a standard Transformer, each position can attend to all positions. In an autoregressive (causal) Transformer, a lower-triangular mask is applied so position i can only attend to positions ≤ i:

Causal Self-Attention
\[ \text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}} + M\right) V \] where \(M_{ij} = -\infty\) if \(j > i\) (mask out future positions), \(0\) otherwise.

📝Summary — The Big Picture
Everything you need to remember from Day 5

What Autoregressive Models Do

1️⃣

Define an Ordering

Choose an ordering for variables: x₁, x₂, …, xₙ (e.g., raster scan for images, left-to-right for text).

2️⃣

Factorize the Joint

Express joint as a product of conditionals via the chain rule: P(X) = ∏ P(xᵢ | x<ᵢ).

3️⃣

Train with MLE

Maximize log-likelihood of training data — equivalent to minimizing KL divergence from the true data distribution.

4️⃣

Sample Recursively

Generate x₁ first, then x₂|x₁, then x₃|x₁,x₂, and so on. Always sequential at generation time.

Key Properties

✅

Tractable Likelihoods: Unlike GANs or VAEs, autoregressive models give exact log-likelihood — great for comparing models and data compression.

✅

Flexible: Extends naturally to multi-class, multi-dimensional, and structured data. Works for text, images, audio, video.

✅

Efficient Training: Modern architectures (PixelCNN, Transformers) allow parallelized training with teacher forcing.

Key Limitations

  • ⚠️ No global latent features: No natural way to extract a compact semantic representation of the data.
  • ⚠️ Not suited for clustering: Not designed for unsupervised feature learning or clustering tasks.
  • ⚠️ Ordering sensitivity: The choice of variable ordering can significantly affect model performance.
  • ⚠️ Sequential generation: Always sequential at inference time — cannot be fully parallelized.

The Appendix Extras: Learning from Histograms

📊 Learning by Counting (Histogram Approach) ▼

The simplest generative model: estimate Pdata by counting frequencies in training data. For a k-valued variable: pi = (# times value i appears) / (total data points).

Problem: learned histogram = training data distribution → poor generalization. The model just memorizes the training set.

Solution: Parameterized distributions (e.g., Gaussian, mixture models, neural networks) that can generalize beyond training examples — the key motivation for all the models in this lecture.

What is the chain rule factorization for autoregressive models?
👆 Click to flip
\[ p(x_1,\ldots,x_n) = \prod_{i=1}^{n} p(x_i \mid x_{<i}) \] Each element is predicted conditioned on all previous elements.
What is the parameter count of FVSBN for n variables?
👆 Click to flip
n(n+1)/2 — quadratic in n. For MNIST (n=784): ~307,000 parameters. Each variable xᵢ has its own weight vector of size i.
What is the key innovation of NADE over FVSBN?
👆 Click to flip
Shared weight matrix W across all conditionals. Instead of separate weights per variable, all conditionals share the same W, drastically reducing parameters and improving generalization.
How does RNADE handle continuous data?
👆 Click to flip
Uses a Mixture of Gaussians: \[ p(x_i \mid x_{<i}) = \sum_k \pi_k \mathcal{N}(x_i \mid \mu_k, \sigma_k^2) \] Parameters (πₖ, μₖ, σₖ) are output by a neural network.
What order does PixelRNN generate pixels in?
👆 Click to flip
Raster-scan order: left-to-right, top-to-bottom (like reading text). Each pixel is conditioned on all pixels above it and to its left.
What is the blind spot problem in PixelCNN?
👆 Click to flip
Masked convolutions prevent seeing pixels to the right and below. After stacking layers, some valid context is always missing from the receptive field, causing artifacts. Fixed by PixelSNAIL with self-attention.
What makes WaveNet able to capture long-range audio dependencies?
👆 Click to flip
Dilated (atrous) convolutions with exponentially growing dilation rates (1, 2, 4, 8, …, 512). This gives a receptive field of 1024+ samples with only ~10 layers.
Why does minimizing KL divergence equal maximizing log-likelihood?
👆 Click to flip
D(P_data ‖ P_θ) = H(P_data) − E[log P_θ(x)]. H(P_data) is constant w.r.t. θ, so minimizing KL ≡ maximizing E[log P_θ(x)] ≡ maximizing log-likelihood!
