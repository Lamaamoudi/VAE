📖 The Big Picture

This lecture takes the Transformer architecture from Day 12 — built for NLP sequences — and applies it directly to images. The central question: can we treat an image the way we treat a sentence?

Answer: Yes! Chop the image into fixed-size patches, treat each patch as a "word", and feed them into a standard Transformer encoder. This is ViT — Vision Transformer (Dosovitskiy et al., ICLR 2021).

ViT Vision Transformer

Patches → Linear projection → [CLS] token → Transformer encoder → classification

DeiT Distillation

Train ViT student to mimic CNN teacher — better with less data

Swin Hierarchical ViT

Windowed attention + patch merging → multi-scale features, linear complexity

DETR Object Detection

End-to-end detection — no anchors, no NMS, Hungarian matching

ConvNeXt Modern CNN

CNN redesigned using Transformer design principles — best of both worlds

🧭 Why Move Beyond CNNs?

CNN Limitations

  • Strong local inductive bias — only sees nearby pixels
  • No direct modeling of global context
  • Receptive field grows slowly layer by layer
  • Translation equivariance is hardwired — less flexible

ViT Advantages

  • Self-attention captures global relationships from layer 1
  • Minimal inductive bias — model learns from data
  • Unified framework bridging vision & language
  • Matches or exceeds CNN accuracy on large datasets

🔢 Core Equations

Patch Embedding

Tokenize image into N patches, project each to D dims
$$z_0 = \left[\mathbf{x}_{\text{class}};\; x_p^1 E;\; x_p^2 E;\; \ldots;\; x_p^N E\right] + E_{\text{pos}}$$
  • $x_p^i \in \mathbb{R}^{P^2 C}$ — flattened $i$-th patch
  • $E \in \mathbb{R}^{(P^2 C) \times D}$ — learned linear projection matrix
  • $\mathbf{x}_{\text{class}} \in \mathbb{R}^D$ — the special learnable [CLS] token
  • $E_{\text{pos}} \in \mathbb{R}^{(N+1) \times D}$ — learned positional embeddings

Each Transformer Block (layer $\ell$)

Multi-Head Self-Attention sublayer
$$z'_\ell = \text{MSA}\!\left(\text{LN}(z_{\ell-1})\right) + z_{\ell-1}$$
MLP sublayer
$$z_\ell = \text{MLP}\!\left(\text{LN}(z'_\ell)\right) + z'_\ell$$
Classification (from [CLS] output of final layer L)
$$y = \text{LN}(z_L^0)$$

🧩 Patch Embedding in Detail

Patch embedding is the crucial first step that converts a 2D image into a 1D sequence of tokens compatible with the Transformer.

📏 The Math of Patch Embedding

Number of patches
$$N = \frac{H \times W}{P^2}$$

For a 224×224 image with P=16: $N = \frac{224 \times 224}{16^2} = \frac{50176}{256} = 196$ patches.

Each flattened patch (before projection)
$$x_p^i \in \mathbb{R}^{P^2 \cdot C} = \mathbb{R}^{16^2 \times 3} = \mathbb{R}^{768}$$
After linear projection
$$z_i = x_p^i \cdot E, \quad E \in \mathbb{R}^{768 \times D}, \quad z_i \in \mathbb{R}^{D}$$

Typical $D$ values: 768 (ViT-Base), 1024 (ViT-Large), 1280 (ViT-Huge).

⚙️ CNN Stem vs Linear Projection

Using a plain linear projection (equivalent to a single non-overlapping convolution with stride=P) has a subtle problem:

Problem: Large stride (P=16) causes optimization instability during training — gradients struggle to flow at the beginning, and the loss can spike or diverge.

Solution: Convolutional Preprocessing Stem

Instead of directly doing a single P×P projection, many ViT variants (e.g., CvT, EfficientViT) use a small CNN stem — a few regular convolutions with small kernel sizes and modest strides — before the patch embedding. This:

  • Stabilizes optimization (gradients flow better)
  • Captures low-level features (edges, textures) that a single linear projection misses
  • Often improves accuracy and convergence speed

📍 Positional Embeddings (Learned)

Unlike the original Transformer (which used sinusoidal encodings), ViT uses learned positional embeddings — one D-dim vector per position, trained from scratch along with the model.

$$\tilde{z}_i = z_i + e_{\text{pos}}^{(i)}, \quad e_{\text{pos}}^{(i)} \in \mathbb{R}^D \text{ (learned)}$$
What the model learns: The position embeddings develop a grid-like similarity structure — nearby patches have similar positional embeddings, and patches in the same row/column cluster together. The model discovers 2D spatial structure entirely from the data!

The [CLS] Token

Borrowed from BERT (NLP). It is a special learnable D-dimensional vector prepended to the sequence before the Transformer. After passing through all encoder blocks, the [CLS] output is a global summary of the entire image, used for classification.

  • The [CLS] token attends to all patch tokens via self-attention in every layer
  • By the final layer, it has aggregated information from the entire image
  • A linear layer (MLP head) maps the [CLS] output to class logits

🔬 What Does ViT Actually Learn?

Opening the "black box" of ViT reveals fascinating and interpretable internal representations.

📐 Position Embedding Similarity

The lecture shows a striking visualization: for a 7×7 patch grid (49 patches from a 224×224 image with P=32), the cosine similarity between each pair of position embeddings forms a clear 2D grid pattern.

  • Each cell shows: how similar is position embedding $e_{\text{pos}}^{(i)}$ to each other position embedding?
  • Nearby patches → high cosine similarity (warm colors)
  • Distant patches → low similarity (cool colors)

Key insight: Even though ViT treats the image as a 1D sequence, its learned positional embeddings encode 2D neighbourhood relationships. The model figures out "which patches are next to each other" by itself.

🌐 Attention Distance vs Network Depth

The right plot in the lecture (ViT-L/16 "mean attention distance" vs layer depth) reveals how attention heads specialize:

1
Lower layers: BOTH local AND global attention heads

In early Transformer blocks, some heads attend to nearby patches (small mean attention distance) and others attend globally (large mean distance). This mixture of local and global processing at early layers is very different from CNNs, where only local processing happens at first.

2
Higher layers: ONLY global attention heads

In deeper layers, almost all attention heads attend globally — the mean attention distance is large (close to the image size in pixels). The model integrates global context in later layers, building a holistic representation for classification.

3
Contrast with CNNs

A CNN starts with very small receptive fields (3×3 or 5×5 convolutions) and can only access distant regions in deep layers via the accumulation of local operations. ViT, by contrast, can attend globally from the very first layer — a fundamentally different information flow.

📊 Attention Distance by Layer (ViT-L/16, schematic)

All heads converge to large attention distances in later layers — globally aware representations.

🎓 Improving ViT: DeiT & Knowledge Distillation

ViT needs massive datasets (ImageNet-21k, JFT-300M) to work well. DeiT (Data-efficient Image Transformers, Touvron et al., ICML 2021) fixes this by training ViT on just ImageNet using knowledge distillation from a CNN teacher.

👩‍🏫 How Knowledge Distillation Works

1
Step 1: Train a Teacher CNN

Train a strong CNN (e.g., ResNet, EfficientNet) on ImageNet using standard cross-entropy loss against ground-truth labels. The teacher learns to output soft probabilities like P(cat) = 0.9, P(dog) = 0.1.

These soft probabilities carry richer information than hard one-hot labels — they encode the teacher's uncertainty and "near misses".

2
Step 2: Train a Student ViT with Two Losses

The student ViT is trained with a combined loss:

DeiT training objective
$$\mathcal{L}_{\text{DeiT}} = (1-\alpha)\,\mathcal{L}_{\text{CE}}(y_{\text{student}}, y_{\text{GT}}) + \alpha\,\mathcal{L}_{\text{KL}}(y_{\text{student}}, y_{\text{teacher}})$$
  • $\mathcal{L}_{\text{CE}}$ — Cross-entropy against ground-truth label (e.g., "Dog")
  • $\mathcal{L}_{\text{KL}}$ — KL divergence against teacher's soft predictions (e.g., P(cat)=0.1, P(dog)=0.9)
  • The teacher's soft labels teach the student about inter-class relationships and model uncertainty
3
Distillation Token (DeiT innovation)

DeiT adds a second special token alongside [CLS]: the distillation token. It is trained to match the teacher's output (via KL divergence), while [CLS] is trained for the ground-truth label (via cross-entropy). At inference, both are averaged for better predictions.

🪟 Swin Transformer: Hierarchical ViT

Liu et al., CVPR 2021 (arxiv 2103.14030). Swin answers the question: can we make ViT hierarchical like a CNN?

Core idea: Instead of global self-attention (quadratic cost), Swin uses local windowed self-attention with shifted windows between layers. Tokens are merged (like pooling) as you go deeper, building a feature pyramid.

Each stage doubles the channels and halves the spatial resolution — exactly like a CNN feature pyramid! This enables Swin to be used as a backbone for detection and segmentation.

🔑 Windowed Self-Attention with Shifting

W
Window Multi-head Self-Attention (W-MSA)

Divide the feature map into non-overlapping windows of M×M tokens. Run self-attention independently within each window. Complexity:

$$\Omega(\text{W-MSA}) = 4hwC^2 + 2M^2 hwC$$

Linear in $hw$ (image size) — unlike global attention which is quadratic $O((hw)^2)$. For M=7 and large images, this is dramatically faster.

S
Shifted Window Multi-head Self-Attention (SW-MSA)

In the next layer, shift the window partition by (M/2, M/2). This creates new windows that cross the boundaries of the previous ones — enabling cross-window connections without extra cost.

Every two consecutive layers form a W-MSA / SW-MSA pair, allowing tokens in adjacent windows to interact every two layers.

M
Patch Merging (like pooling)

Between stages, 2×2 patches are concatenated and projected — halving spatial resolution and doubling channels. This is the hierarchical "downsampling" that makes Swin work like a CNN pyramid.

$$\text{Merge}(z) : (H \times W \times C) \to \left(\frac{H}{2} \times \frac{W}{2} \times 2C\right)$$

📦 DETR: End-to-End Object Detection with Transformers

Carion et al., ECCV 2020. DETR completely reimagines object detection — no anchors, no NMS, no region proposals. Just a Transformer.

🔄 Traditional Detection Pipeline vs DETR

Traditional (e.g., Faster R-CNN)

  • Hand-crafted anchor boxes
  • Region Proposal Network (RPN)
  • Non-Maximum Suppression (NMS)
  • Complex multi-stage pipeline
  • Many hyperparameters to tune
  • Not end-to-end trainable

DETR

  • No anchors, no proposals
  • No NMS needed
  • Simple, unified pipeline
  • Fully end-to-end trainable

🔑 Key Concepts in DETR

Q
Object Queries (N learned vectors)

DETR uses a fixed set of $N$ learnable vectors called object queries (e.g., N=100). Each query "asks" the decoder to find one object in the image. The decoder uses cross-attention to look at encoder features and produce one prediction per query.

Queries replace anchors and region proposals. They are learned from scratch and implicitly specialize for different object sizes and positions.

H
Hungarian Algorithm (Bipartite Matching Loss)

Since DETR produces N predictions but there may be fewer ground-truth objects, it uses the Hungarian algorithm to find the optimal 1-to-1 assignment between predicted boxes and ground-truth boxes (minimizing a combined class + box cost).

Unmatched predictions are assigned the "no object" class (∅). This ensures no duplicate detections and eliminates the need for NMS.

Matching cost (simplified)
$$\mathcal{L}_{\text{match}} = -\mathbb{1}_{c_i \ne \varnothing}\,\hat{p}_{\sigma(i)}(c_i) + \mathbb{1}_{c_i \ne \varnothing}\,\mathcal{L}_{\text{box}}(b_i, \hat{b}_{\sigma(i)})$$
E
Encoder: Global Self-Attention over CNN Features

The CNN backbone extracts a spatial feature map. Flattened into a sequence with positional encodings added, this passes through a standard Transformer encoder. Every spatial location can attend to every other location globally — enabling global context for detection.

D
Decoder: Object Queries attend to Encoder

Each object query attends to the encoder output via cross-attention. The decoder also uses self-attention among the N queries, allowing them to avoid predicting the same object twice. Each query's final vector is passed to two FFN heads: one for class, one for box coordinates.

🔁 ConvNeXt: A ConvNet for the 2020s

Liu et al., CVPR 2022. ConvNeXt asks: what if we modernize a CNN using every design principle from Transformers? The result is a pure CNN that matches or beats ViT and Swin.

Key message: ViTs don't win because of self-attention per se — they win because of better design choices. Many of those choices can be applied to CNNs too, making CNNs competitive again.

🛠️ Design Changes: ResNet → ConvNeXt

1
Hierarchical design similar to Swin Transformer

Use a 4-stage hierarchical design with patch-stem + downsampling layers between stages, just like Swin. Stage ratios: 3:3:9:3 blocks (matching Swin's 2:2:6:2).

2
Depthwise Separable Convolutions (large kernel)

Replace 3×3 grouped convolutions with depthwise separable 7×7 convolutions. The large kernel size mimics the large "receptive field" of self-attention within a window (like Swin's 7×7 window). Depthwise means each channel is convolved separately — fewer parameters, same expressivity.

3
Layer Normalization instead of Batch Normalization

ViT/Swin use LayerNorm. ConvNeXt adopts LayerNorm (which normalizes over the channel dimension) instead of BatchNorm — more stable for small batch sizes and better for transfer learning.

4
GELU Activation (from Transformers)

Replace ReLU with GELU (Gaussian Error Linear Unit) — used in all Transformer models. GELU is smoother than ReLU and slightly improves accuracy.

$$\text{GELU}(x) = x \cdot \Phi(x) \approx 0.5x\left(1 + \tanh\!\left(\sqrt{\frac{2}{\pi}}(x + 0.044715x^3)\right)\right)$$
5
Inverted Bottleneck + Fewer Activations

Like Transformers' MLP (expand × 4, then contract), ConvNeXt uses an inverted bottleneck: expand channels 4× in the pointwise layers, with only one activation function (not after every layer like ResNet). This simplifies the block.

6
Global Average Pooling

Uses global average pooling before the classification head — the same approach used in Transformer models, replacing the fully-connected layers in traditional CNNs.

⚠️ Limitations & Future Directions

ViTs are powerful but come with real trade-offs that active research is working to address.

📊 Current Limitations

1
Data Hunger — Needs Large-Scale Pre-training

ViTs lack the strong inductive biases of CNNs (locality, translation equivariance). Without these priors, the model must learn them from data — which requires enormous datasets. On ImageNet alone (~1M images), ViT underperforms ResNet. It only wins with ImageNet-21k (14M) or JFT-300M (300M images).

Mitigation: Knowledge distillation (DeiT), data augmentation (RandAugment, Mixup, CutMix), self-supervised pre-training (MAE, DINO).

2
Quadratic Computational Complexity

Self-attention over $N$ tokens costs $O(N^2)$ — for a 224×224 image with P=16, $N=196$ tokens. But for higher resolution images (e.g., 512×512 with P=16, $N=1024$), the cost grows quadratically. Doubling resolution → 4× more compute for attention.

$$\text{Attention cost} \propto N^2 = \left(\frac{H \cdot W}{P^2}\right)^2$$

Mitigations: Windowed attention (Swin), linear attention, sparse attention, FlashAttention.

3
Optimization Sensitivity

ViTs are sensitive to hyperparameter choices (learning rate, warmup schedule, weight decay, dropout). They require:

  • Learning rate warmup (typically 10k steps)
  • Cosine learning rate decay
  • Strong regularization (DropPath, Label Smoothing, RandAugment)
  • Large batch sizes

CNNs are much more forgiving with default training settings.

4
Interpretability Limitations

Attention maps can highlight which image regions the model focuses on — a form of visual interpretability. However:

  • Attention maps don't always align with human-interpretable features
  • They may focus on background or irrelevant regions
  • Multiple heads attend to different things — which one to trust?
  • High-level attention maps may become very diffuse
5
Deployment Challenges

ViTs are resource-intensive:

  • Larger memory footprint than CNNs of similar accuracy
  • Less hardware-friendly — specialized hardware optimizes for convolutions
  • Not suitable for edge devices, IoT, or real-time applications without modification

Active research: model pruning, quantization, token pruning (removing uninformative tokens), lightweight ViT designs (MobileViT, TinyViT, EfficientViT).

🔮 Future Directions

  • Efficient attention mechanisms: Linear attention, sparse attention, FlashAttention, and other $O(N)$ alternatives to quadratic self-attention
  • CNN + Transformer hybrids: Leverage CNN's data efficiency and locality while gaining Transformer's global context (e.g., CMT, MobileViT, CoAtNet)
  • Lightweight ViTs for edge: Pruning, quantization (INT8/INT4), knowledge distillation to tiny student models for mobile deployment
  • Self-supervised pre-training: MAE (Masked Autoencoders), DINO, SimMIM — train ViT without labels to reduce data requirements
  • Unified multimodal models: ViT enables the same architecture for images, text, and video (e.g., ViT-based CLIP, Flamingo, GPT-4V)
