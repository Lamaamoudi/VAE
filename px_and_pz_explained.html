p(x) — Probability of an image
"How likely is this image to naturally exist?"

x is an image — say a 28×28 grid of pixels (like MNIST digits). p(x) asks: among all the billions of possible pixel combinations, how likely is this particular one to be a real, meaningful image?

🎲 Think of it this way

If you randomly set every pixel to a random color, you'd almost certainly get noise — not a real digit. The space of all possible images is huge. Real, meaningful images occupy a tiny slice of it. p(x) is high for images that look like real data, and near-zero for random noise.

🎯 Why we care

Training a VAE = maximizing p(x) for all images in our dataset. We want the model to say "yes, these images make perfect sense!" But computing p(x) requires that impossible integral — so we use the ELBO instead.

🎛️
p(z) — Probability of a latent code
"What should z look like before we see any image?"

z is the hidden description — a short list of numbers that summarizes an image. p(z) is our assumption about what z should look like in general, before looking at any specific image.

🧠 We CHOOSE p(z) — it's a design decision

In a VAE, we decide that p(z) = N(0, 1) — a standard normal distribution. This means we expect z values to be numbers roughly between -3 and 3, centered at zero. We pick this because it's simple and easy to sample from.

Here's what z might look like for a cat image — a vector of numbers:

These numbers are all roughly between -2 and 2 — that's what p(z) = N(0,1) means.

p(z) = standard normal bell curve. Values near 0 are common, extremes are rare.

💡 Why standard normal specifically?

Because at generation time we want to just sample a random z and decode it into a new image. If the latent space follows N(0,1), we can do exactly that — just pick random numbers from a standard normal and feed them to the decoder!
