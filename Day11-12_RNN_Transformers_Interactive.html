🧭 The Big Picture

Here's how everything connects:

1
Sequential data needs memory → RNNs invented

Traditional feedforward networks ignore order. To handle text, speech, and time series you need the model to "remember" previous steps. RNNs achieve this with a hidden state passed forward through time.

2
Seq2Seq: encoder compresses, decoder generates

For tasks like machine translation, a two-RNN setup emerged: the encoder reads the entire input and compresses it into a single context vector. The decoder uses that vector to generate output word by word.

3
Bottleneck problem → Attention mechanism added to RNNs

A single context vector can't capture everything for long inputs. Attention solves this by letting the decoder dynamically "look back" at all encoder hidden states instead of just the last one.

4
RNNs are sequential → can't parallelize → Transformers born

Even with attention, RNNs must process tokens one at a time. Vaswani et al. (2017) asked: what if we use only attention, with no recurrence at all? The result was the Transformer — fully parallelizable and far more scalable.

🔁 What is an RNN?

Definition: An RNN is a neural network with a looping mechanism — the output at each time step feeds back as input to the next step. This loop gives the network a form of memory.

Core Recurrence Equation

At every time step $t$, the hidden state is updated as:

Hidden State Update
$$h_t = \sigma\!\left(W_h\, h_{t-1} + W_x\, x_t + b\right)$$
  • $h_t$ — hidden state at time $t$ (the "memory")
  • $h_{t-1}$ — hidden state from the previous step
  • $x_t$ — input at time $t$
  • $W_h$ — weight matrix for the hidden state (recurrent weights)
  • $W_x$ — weight matrix for the input
  • $b$ — bias vector
  • $\sigma$ — activation function (e.g., tanh or ReLU)
Key insight: The same weights $W_h$ and $W_x$ are shared across all time steps. This is what keeps the model size constant regardless of sequence length.

⚙️ Training: Backpropagation Through Time (BPTT)

Training an RNN uses BPTT — the network is unrolled over time and gradients are backpropagated through each time step.

Gradient chain through time
$$\frac{\partial \mathcal{L}}{\partial W_h} = \sum_{t=1}^{T} \frac{\partial \mathcal{L}_t}{\partial W_h}$$

The chain rule must be applied across all $T$ time steps, which creates a long product of Jacobians — the source of gradient problems.

✅ Advantages

  • Processes any length input sequence
  • Model size constant regardless of sequence length
  • Same weights at every time step (symmetry)
  • Theoretically captures information from many steps back

❌ Disadvantages

  • Recurrent computation is sequential — slow
  • In practice, only ~10 steps of real memory
  • Vanishing gradients: gradients shrink to 0 over time
  • Exploding gradients: gradients blow up exponentially

🔀 Sequence-to-Sequence Models

Many real-world tasks require mapping one sequence to another (e.g., English → French, question → answer). The Encoder–Decoder framework handles this.

🗜️ Encoder

  • Processes the entire input sequence one token at a time
  • Produces a series of hidden states $h_1, h_2, \ldots, h_T$
  • The final hidden state $h_T$ becomes the context vector $c$
  • This context vector is supposed to summarize the entire input
Context Vector (no attention)
$$c = h_T^{\text{encoder}}$$

🗣️ Decoder

  • Initialized with the encoder's context vector $c$
  • Generates output tokens one at a time
  • Each step uses: the context vector, its own previous hidden state, and the previous output token
  • Stops when it generates a special end token
Decoder step
$$s_t = f(s_{t-1},\; y_{t-1},\; c)$$ $$y_t = \text{softmax}(W_s\, s_t)$$

where $s_t$ is the decoder hidden state and $y_t$ is the output token at step $t$.

⚠️ The Bottleneck Problem — With Empirical Evidence

Problem: Compressing an entire long sentence into a single fixed-size vector loses information — especially for long sequences. What if T = 1000?

The lecture shows a BLEU score graph (Cho et al., 2014) that makes this concrete:

The solution? Attention — the decoder can look at all encoder hidden states, not just the last one.

🎯 Attention in Seq2Seq

Instead of bottlenecking through a single context vector, attention lets the decoder dynamically focus on different parts of the input at each output step.

Core idea: At each decoder time step $t$, compute a different context vector $c_t$ by taking a weighted sum of all encoder hidden states.

$$h_i = \left[\vec{h}_i\, ;\, \overleftarrow{h}_i\right]$$

Each annotation $h_i$ captures context from both directions — what came before and after position $i$. This gives richer encoder representations for the attention mechanism to work with.

🧮 How Attention Works — Step by Step

1
Compute alignment scores (using an MLP)

For each encoder hidden state $h_i$ and the current decoder state $s_{t-1}$, compute a scalar compatibility score using a small learned MLP $f_{\text{att}}$:

$$e_{t,i} = f_{\text{att}}(s_{t-1},\; h_i)$$

This MLP is trained end-to-end — no separate supervision needed (see below).

2
Normalize with Softmax → attention weights
$$\alpha_{t,i} = \frac{\exp(e_{t,i})}{\sum_{j=1}^{T}\exp(e_{t,j})}, \quad 0 < \alpha_{t,i} < 1,\quad \sum_i \alpha_{t,i} = 1$$

The weights $\alpha_{t,i}$ sum to 1 across all encoder positions $i$. They tell the decoder how much to "look at" each encoder state at this decoding step.

3
Weighted sum → dynamic context vector
$$c_t = \sum_{i=1}^{T} \alpha_{t,i}\, h_i$$

Context vector $c_t$ is a linear combination of hidden states, weighted by attention. Different at each decoder step — it "attends" to relevant input positions.

Concrete example (from the lecture): translating "we are eating bread" → Spanish. When generating "estamos" (= "we are"), the model attends mostly to positions 1 and 2: $\alpha_{1,1} = \alpha_{1,2} \approx 0.45$, $\alpha_{1,3} = \alpha_{1,4} \approx 0.05$. When generating "comiendo" (= "eating"): $\alpha_{2,3} \approx 0.8$, $\alpha_{2,2} \approx 0.1$, $\alpha_{2,1} = \alpha_{2,4} \approx 0.05$.

4
Generate output token using context
$$s_t = g_U(y_{t-1},\; s_{t-1},\; c_t)$$

The decoder uses its previous state $s_{t-1}$, the previous output $y_{t-1}$, and the dynamically computed context $c_t$ to produce the next token.

🔑 Key insight from the lecture: This is fully differentiable. There is no explicit supervision on the attention weights — they are learned purely through backpropagation of the translation loss. The model discovers what to attend to automatically!

📐 Attention Summary: Tensor Shapes

The full attention mechanism for image captioning (from the lecture's summary slide) with explicit shapes:

Inputs
$$\mathbf{z} \in \mathbb{R}^{H \times W \times D} \quad \text{(spatial CNN features)}, \quad \mathbf{h} \in \mathbb{R}^{D} \quad \text{(decoder query)}$$
Alignment scores (one per spatial location)
$$e_{i,j} = f_{\text{att}}(\mathbf{h},\; \mathbf{z}_{i,j}), \quad \mathbf{e} \in \mathbb{R}^{H \times W}$$
Attention weights (softmax over spatial grid)
$$\mathbf{a} = \text{softmax}(\mathbf{e}), \quad \mathbf{a} \in \mathbb{R}^{H \times W}$$
Context vector (weighted sum over spatial locations)
$$\mathbf{c} = \sum_{i,j} a_{i,j}\, \mathbf{z}_{i,j}, \quad \mathbf{c} \in \mathbb{R}^{D}$$

🖼️ Image Captioning: CNN + RNN Architecture

Before attention, image captioning used a simple CNN → RNN pipeline. The CNN (e.g., VGG or ResNet) extracts a visual feature vector $v_i$, which seeds the RNN decoder:

RNN decoder for image captioning
$$h_t = \tanh(W_{hh}\, h_{t-1} + W_{hx}\, x_t + b_h)$$ $$y_t = \text{softmax}(W_{oh}\, h_t + b_o)$$
  • $W_{hh}$ — hidden-to-hidden weight matrix (recurrent connections)
  • $W_{hx}$ — input-to-hidden weight matrix (processes word embeddings)
  • $W_{oh}$ — hidden-to-output weight matrix (produces word probabilities)
  • The CNN feature $v_i$ initializes the hidden state $h_0$ of the RNN
  • Input sequence: [START] → "straw" → "hat" → … The RNN generates one word per step until END

With Attention: "Show, Attend and Tell" (Xu et al., ICML 2015)

  • CNN extracts a grid of feature vectors $h_{i,j}$ (not just one vector)
  • At each caption word generation step, the decoder attends to different spatial regions
  • Generating "cat" → attention focuses on the cat region; "sitting" → focuses on the body; "outside" → focuses on the background
  • Each decoder step uses a different context vector $c_t = \sum_{i,j} a_{t,i,j}\, h_{i,j}$
Results from the paper: For "A dog is standing on a hardwood floor" — the word "dog" gets high attention on the dog region; for "A stop sign on a road with a mountain in the background" — "stop" lights up the sign, "mountain" lights up the background. The model learns to look at the right place automatically!

⚠️ Limitations of RNNs

Bottom line: RNNs opened the door to sequence modeling but suffer from fundamental architectural limitations that motivated the invention of Transformers.

1️⃣ Long-term Dependency Problem

Basic RNNs have difficulty capturing dependencies in sequences longer than ~10 steps.

Why? Vanishing Gradients

During BPTT, gradients must be multiplied together across many time steps:

$$\frac{\partial h_t}{\partial h_1} = \prod_{k=1}^{t-1} \frac{\partial h_{k+1}}{\partial h_k} = \prod_{k=1}^{t-1} W_h^\top \cdot \text{diag}(\sigma'(h_k))$$
  • If the entries of $W_h$ are $< 1$, the product shrinks toward 0 → vanishing gradient → early steps contribute nothing to learning
  • If the entries of $W_h$ are $> 1$, the product explodes → exploding gradient → numerical instability
Historical note: Hochreiter & Schmidhuber (1997) invented LSTMs specifically to solve the vanishing gradient problem via gating mechanisms. Cho et al. (2014) later proposed the simpler GRU.

2️⃣ Sequential Computation — No Parallelism

An RNN must process $x_1$ before $x_2$, before $x_3$, etc. You cannot parallelize across time steps within a sequence.

  • Makes training on long sequences very slow
  • Modern GPUs excel at parallel matrix multiplications — RNNs barely use this
  • Transformers, in contrast, process all positions simultaneously

3️⃣ Alternatives that Emerged

ArchitectureImprovementStatus
LSTMGating → solves vanishing gradientsStill used for some tasks
GRUSimpler gating than LSTMStill used, fewer params
TransformerFull attention, no recurrence, parallelizableDominant architecture today
LLMs (GPT, BERT, etc.)Transformer scaled to billions of paramsState of the art

Day 12 💡 Why Transformers?

The 2017 paper "Attention Is All You Need" (Vaswani et al., NeurIPS 2017) introduced the Transformer — arguably the most impactful neural network architecture of the past decade.

🚫 Problems with RNNs & CNNs

ProblemRNNsCNNs

✅ Transformers Address All of These

What Transformers provide

  • Global context: every token attends to every other token
  • Full parallelization: all positions computed simultaneously
  • Scalability: easily stacked into deep models
  • Flexibility: works for text, images, audio, code…

Trade-offs

  • Quadratic complexity in sequence length: $O(T^2)$ attention
  • No built-in position awareness (must be added)
  • Require large amounts of data and compute

🧩 General Attention Layer

Before self-attention, let's understand the general attention formulation used in the Transformer.

The Three Ingredients: Queries, Keys, Values

SymbolRoleAnalogy
$Q$ — QueryWhat we're looking forSearch query
$K$ — KeyWhat each item offersDocument index entry
$V$ — ValueThe actual content to retrieveDocument content

Scaled Dot-Product Attention

Core Attention Formula
$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{D}}\right) V$$
  • $Q \in \mathbb{R}^{N_q \times D}$, $K \in \mathbb{R}^{N \times D}$, $V \in \mathbb{R}^{N \times D_v}$
  • $QK^\top$ computes pairwise dot-product similarities between queries and keys
  • Divide by $\sqrt{D}$ — why? See below
  • Softmax normalizes scores into a probability distribution (attention weights)
  • Multiply by $V$ to get a weighted sum of values

📐 Why Scale by $\sqrt{D}$?

The problem: As feature dimension $D$ grows, the dot products $QK^\top$ grow in magnitude (larger variance). This causes the softmax to become extremely peaked — almost all weight goes to one position.
  • If queries and keys are IID with mean 0 and variance 1, their dot product has variance $D$
  • Dividing by $\sqrt{D}$ brings variance back to 1
  • This keeps the softmax in a region with healthy gradients — not too concentrated, not too uniform
$$\text{Var}\!\left(\frac{q \cdot k}{\sqrt{D}}\right) = \frac{\text{Var}(q \cdot k)}{D} = \frac{D}{D} = 1$$

🔄 Multiple Queries & Separate Projections

The attention layer can be made more expressive in two key ways:

1
Multiple query vectors simultaneously

Instead of a single query, use $N_q$ queries at once. Each query attends independently and produces its own context vector. This lets the model extract different types of information from the same input in parallel.

2
Separate FC layers for Q, K, V

Instead of using the raw input vectors directly as queries, keys, and values, we apply separate learned linear (FC) transformations:

$$Q = X W_Q, \quad K = X W_K, \quad V = X W_V$$

This is crucial: different weight matrices $W_Q$, $W_K$, $W_V$ allow the model to learn different representations for attending vs. aggregating. Input/output dimensions can also differ.

🔁 Permutation Invariance of Attention

The attention operation is permutation invariant — if you shuffle the input vectors, you get the same output vectors (just shuffled the same way). This means:

  • The model doesn't inherently know which position came first
  • For images: reshape $H \times W$ spatial grid into $N = H \times W$ feature vectors — order doesn't matter to attention
  • For sequences: you must add positional encoding (covered in the next section)

🔍 Self-Attention Layer

Key difference from general attention: In self-attention, the queries, keys, and values all come from the same input sequence. There's no separate "source" and "target."
Self-Attention
$$Q = X W_Q, \quad K = X W_K, \quad V = X W_V$$ $$\text{SelfAttention}(X) = \text{softmax}\!\left(\frac{X W_Q (X W_K)^\top}{\sqrt{D_k}}\right) X W_V$$

Every position in the sequence can attend to every other position — capturing global context in a single layer.

⚖️ General Attention vs. Self-Attention

FeatureGeneral AttentionSelf-Attention
Q, K, V originsFrom separate source & targetFrom the same input sequence
Use caseEncoder → Decoder cross-attentionEncoder or Decoder internal
Information flowAcross two representationsWithin a single representation
ExampleTranslation: decoder attends to encoderBERT: each word attends to all others

🙈 Masked Self-Attention

Used in the decoder during training (and inference) to prevent "cheating" — looking at future tokens.

  • Set the alignment scores of all future positions to $-\infty$ before the softmax
  • After softmax, $\exp(-\infty) = 0$ → zero attention weight on future tokens
  • Ensures each position $i$ only attends to positions $\leq i$
Masking
$$e_{ij} = \begin{cases} q_i \cdot k_j / \sqrt{D} & j \leq i \\ -\infty & j > i \end{cases}$$
Intuition: When generating word 3 of a translation, the decoder should only be informed by its own words 1 and 2 — not words 4 or 5 that it hasn't generated yet.

🎭 Multi-Head Attention

Instead of running a single attention operation, run $H$ parallel attention "heads" and concatenate their outputs.

Multi-Head Attention
$$\text{head}_i = \text{Attention}(Q W_Q^{(i)},\; K W_K^{(i)},\; V W_V^{(i)})$$ $$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_H)\, W_O$$

Why multiple heads?

  • Each head can specialize in a different type of relationship (e.g., subject–verb, coreference, syntactic structure)
  • Head 1 might focus on short-range dependencies; head 2 on long-range
  • Concatenating then projecting with $W_O$ merges all perspectives
  • Richer, more expressive representations than any single attention pass
Original paper (Vaswani et al. 2017): Used $H = 8$ heads, model dimension $D = 512$, so each head has dimension $D/H = 64$.

🌐 CNN + Self-Attention — Full Pipeline with Tensor Shapes

The lecture walks through this step-by-step. This is the exact flow shown in slides 22–27:

1
CNN extracts feature map: $C \times H \times W$

A standard CNN processes the input image and outputs a spatial feature tensor of shape $C \times H \times W$, where $C$ is the number of channels and $H \times W$ is the spatial resolution.

2
Three 1×1 Convolutions → Q, K, V (each $C' \times H \times W$)

Three separate $1 \times 1$ convolutions project the feature map into query, key, and value tensors (each of reduced dimension $C'$):

$$Q = \text{Conv}_{1\times1}(F),\quad K = \text{Conv}_{1\times1}(F),\quad V = \text{Conv}_{1\times1}(F) \quad \in \mathbb{R}^{C' \times H \times W}$$
3
Attention weights: $(H \times W) \times (H \times W)$

Reshape Q and K to $(C') \times (HW)$, compute pairwise dot products, then softmax:

$$A = \text{softmax}\!\left(\frac{Q^\top K}{\sqrt{C'}}\right) \in \mathbb{R}^{(HW) \times (HW)}$$

Every spatial position attends to every other spatial position — this is where local CNN features become globally aware.

4
Apply weights to V → $C' \times H \times W$, project → $C \times H \times W$

Multiply attention weights by values, then a final $1 \times 1$ conv brings dimension back to $C$:

$$\text{Out} = \text{Conv}_{1\times1}(A \cdot V) \in \mathbb{R}^{C \times H \times W}$$
5
Residual connection → final output

The self-attention output is added back to the original CNN features (residual connection), just like in the full Transformer:

$$\text{Final} = F + \text{Out} \in \mathbb{R}^{C \times H \times W}$$

This ensures the model only needs to learn the difference from the original features, making training more stable.

Why this matters: CNN alone only has a local receptive field. Self-attention makes every pixel/patch aware of the entire image in a single layer. This hybrid approach is used in models like SAGAN and predates pure Vision Transformers (ViT).

📍 The Positional Encoding Problem

Self-attention is permutation equivariant: shuffle the inputs, get the same outputs shuffled. This means the model has no idea about token order by default.

But order is crucial for language! "The cat ate the fish" ≠ "The fish ate the cat".

Solution: Add a positional encoding vector $p_j$ to each input vector $x_j$ before the attention computation: $$\tilde{x}_j = x_j + p_j$$

〽️ Sinusoidal Positional Encodings

The original Transformer used fixed sinusoidal encodings. For position $j$ and dimension index $k$:

Sinusoidal Positional Encoding (even dims)
$$\text{PE}(j, 2k) = \sin\!\left(\frac{j}{10000^{2k/d}}\right)$$
Sinusoidal Positional Encoding (odd dims)
$$\text{PE}(j, 2k+1) = \cos\!\left(\frac{j}{10000^{2k/d}}\right)$$
  • $j$ = position in the sequence (0, 1, 2, …)
  • $k$ = dimension index within the encoding vector (0 to $d/2 - 1$)
  • $d$ = model dimension (e.g., 512)
  • Each position gets a unique $d$-dimensional encoding

🤔 Why Sinusoidal?

  • Unique per position: Each position gets a different encoding
  • Smooth relative offsets: $\text{PE}(j + \delta)$ can be expressed as a linear function of $\text{PE}(j)$ → the model can learn to reason about relative positions
  • Generalizes to unseen lengths: Since it's a deterministic function, it works for sequences longer than those seen during training
  • No learnable parameters needed

Learned vs. Fixed Positional Encodings

Generalizes beyond training length✅ Yes❌ No

🏗️ The Transformer Block

A single Transformer block consists of two sub-layers, each with a residual connection and layer normalization:

📥
Input: set of token vectors $x_1, \ldots, x_T$ (with positional encoding added)
↓
🔍
Multi-Head Self-Attention + Residual + LayerNorm
Each token attends to all other tokens
↓
🧮
MLP (Feed-Forward Network) + Residual + LayerNorm
Applied independently to each token: 2 FC layers with ReLU
↓
📤
Output: set of refined token vectors $y_1, \ldots, y_T$
Transformer Block (formal)
$$z_i = \text{LayerNorm}\!\left(x_i + \text{MultiHead}(x_i)\right)$$ $$y_i = \text{LayerNorm}\!\left(z_i + \text{MLP}(z_i)\right)$$

📚 Stacking Blocks

The full Transformer stacks $N$ identical blocks. Each block refines the token representations.

  • Original Vaswani et al. (2017): $N = 12$ blocks
  • Hidden size: $D = 512$
  • Attention heads: $H = 6$
Key property: Because attention and MLP operations are applied independently per token, the entire computation is highly parallelizable — GPU-friendly and fast!

🔑 Why Residual Connections & Layer Normalization?

R
Residual Connections: $y = x + f(x)$

Adding the input directly to the output ($x + f(x)$) allows gradients to flow directly through many layers without passing through the transformation $f$. This solves the vanishing gradient problem in deep networks (same idea as ResNets in computer vision).

L
Layer Normalization

Normalizes the activations across the feature dimension (not the batch). This stabilizes training by keeping activations in a healthy range, especially important for deep stacks of Transformer blocks.

$$\text{LayerNorm}(x) = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta$$

🔗 How the Lectures Connect

1
The link: Attention was born in RNNs

Attention was first introduced as an add-on to RNN Seq2Seq models (Bahdanau et al., 2014) to solve the bottleneck problem. The Transformer then took the insight further: what if we used only attention — no RNN at all?

2
Transformer's self-attention is a generalization

In RNN attention: the query comes from the decoder state, keys/values from encoder states (cross-attention). In Transformer self-attention: queries, keys, values all come from the same sequence. Same math, different source.

3
Both lectures build toward LLMs

RNNs → Seq2Seq → Attention → Transformer → LLMs (GPT, BERT, LLaMA). Each step solved problems of the previous. Today's most powerful AI systems are based on stacked Transformer blocks at massive scale.
